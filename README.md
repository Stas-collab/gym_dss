# GymDSS

Тема — вибір спортзалу для студента на основі багатокритеріального аналізу.

## Про що це

Є кілька спортзалів, і треба вибрати найкращий з урахуванням різних факторів — ціна, відстань, обладнання тощо. Кожен критерій має свою вагу і тип (maximize або minimize). Система рахує інтегральну оцінку для кожного спортзалу і видає рейтинг.

Реалізовано два методи: SAW і TOPSIS — можна переключати прямо в інтерфейсі.

## Стек

- Backend: Node.js + Express + MongoDB (Mongoose)
- Frontend: React + Vite

## Як запустити

Потрібен Node.js v22+ і MongoDB на локальній машині.

```bash
# backend
cd backend
npm install
# створити .env файл (див. нижче)
npm run dev
```

```bash
# frontend (в іншому терміналі)
cd frontend
npm install
npm run dev
```

Файл `backend/.env`:

```
MONGO_URI=mongodb://127.0.0.1:27017/dss_gym
PORT=5000
```

Після запуску: http://localhost:5173

## Предметна область

Вибір спортзалу — задача напівструктурована, бо є і кількісні критерії (ціна, відстань) і суб'єктивні (комфорт, атмосфера). Саме тому тут доречно використовувати СППР.

**Альтернативи (A):**

- a1: FitLife — популярний студентський спортзал
- a2: SportMax — великий спортивний комплекс
- a3: EnergyGym — сучасний зал з широким вибором занять
- a4: PowerZone — профільний зал для важкої атлетики
- a5: CityFit — доступний зал у центрі міста

**Критерії (C):**
| Критерій | Тип | Вага |
|----------|-----|------|
| Ціна | minimize | 8 |
| Комфорт | maximize | 6 |
| Відстань | minimize | 5 |
| Обладнання | maximize | 7 |
| Атмосфера | maximize | 4 |

**Матриця оцінок:**
| | Ціна | Комфорт | Відстань | Обладнання | Атмосфера |
|--|------|---------|----------|------------|-----------|
| FitLife | 8 | 6 | 7 | 8 | 7 |
| SportMax | 5 | 7 | 5 | 9 | 6 |
| EnergyGym | 9 | 8 | 8 | 7 | 9 |
| PowerZone | 6 | 5 | 4 | 10 | 6 |
| CityFit | 7 | 9 | 6 | 6 | 10 |

## Методи аналізу

**SAW** — проста зважена сума. Кожне значення нормалізується (мінімакс з урахуванням типу критерію), множиться на вагу, і все підсумовується. Чим вище — тим краще.

**TOPSIS** — метод на основі відстані до ідеального рішення. Будується ідеальна альтернатива A+ (найкраще по кожному критерію) і антиідеальна A−. Переможець — той, хто найближче до A+ і найдальше від A−.

## База даних

Три колекції в MongoDB:

- `alternatives` — спортзали
- `criteria` — критерії з вагами і типами
- `evaluations` — оцінки (зв'язок alternative_id + criterion_id + value)

## API

```
GET  /api/alternatives       — список спортзалів
POST /api/alternatives       — додати
DELETE /api/alternatives/:id — видалити

GET  /api/criteria           — список критеріїв
POST /api/criteria           — додати
DELETE /api/criteria/:id     — видалити

GET  /api/evaluations        — всі оцінки
POST /api/evaluations        — додати або оновити оцінку

POST /api/analyze            — запустити аналіз
     body: { "method": "saw" } або { "method": "topsis" }
```
